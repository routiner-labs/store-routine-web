## 84. 알림 패널 — 최상위 + 벨에 부착

사용자 요청: 알림 창은 항상 모든 창보다 최상위. 지금 알림창이 버튼과 조금 동떨어져 있음.

원인:
- `.nav`가 `position: sticky` → 독립 stacking context 생성. 패널이 nav 내부에 있어 z-index를 올려도 nav 밖 모달(300)/컨펌(500) 위로 못 올라감
- 패널이 `position: fixed`로 사이드바 가장자리(`bottom:72px; left:sidebar+8`)에 고정 → 벨과 동떨어짐

수정된 파일:
- `src/components/NotificationBell.tsx` — 패널/오버레이를 `createPortal(..., document.body)`로 body에 렌더(스택 컨텍스트 탈출). 벨 `ref`의 `getBoundingClientRect`로 위치 계산(`left: 벨 right+8`, `bottom: 벨 하단 정렬`)해 인라인 style로 지정. 열 때 위치 계산(`toggle`)
- `src/components/NotificationBell.module.css` — `.panel` z-index 200→1200(+포탈이라 실제 최상위), 뷰포트 고정/미디어쿼리 위치 계산 제거(인라인 위치로 대체), `.overlay` z-index 99→1199

주요 결정:
- 포탈 + z-index 1200 → 모달/컨펌 위 최상위(토스트 9999는 상단 유지 — 토스트는 '창'이 아님)
- 벨 rect 기준으로 붙여 "동떨어짐" 해소(벨 오른쪽 8px, 하단 정렬)
- 모바일은 기존 전체화면 알림(`notifScreen`, OwnerNav)이라 영향 없음
- 헤드리스 검증: 패널 body 직속·z-index 1200·position fixed·left=벨right+8·bottom=벨하단, 오류 0 / 빌드 성공
- 후속: 패널을 살짝 위로(벨 하단 정렬 + 14px) — `bottom: innerHeight - r.bottom + 14`
