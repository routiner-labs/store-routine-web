- 개별 문서 카드/상세 페이지의 카테고리 뱃지(`catBadge`, 정보 표시용)는 이번 변경과 무관하게 그대로 유지 — 이번에 바뀐 건 "필터링 UI"뿐, 문서 자체의 카테고리 표시 방식은 손대지 않음
- 검증: `tsc --noEmit`·`eslint` 클린, curl로 상단 카테고리 칩이 사라지고 "전체 카테고리" 버튼 라벨이 세부검색 패널에 렌더링됨을 확인

## 94. 카테고리 선택 후 X(초기화) 버튼 위치 수정

사용자 요청: "카테고리 선택되고 나서 x 버튼의 위치가 좀 안맞아"

원인: 카테고리 행이 `.advRow`(`align-items: flex-start`) 안에 `catFilterBtn`과 `dateClear`(22px 원형 X버튼)를 바로 나열해서, 버튼 높이가 다른 두 요소가 상단 기준으로 정렬돼 X버튼이 카테고리 버튼과 세로 중앙이 맞지 않았음. 작성자 행은 `.advSearchWrap`(`align-items: center`)으로 감싸져 있어 문제가 없었음.

수정된 파일:
- `src/app/owner/documents/page.tsx` — 카테고리 행의 `catFilterBtn`+`dateClear`를 작성자 행과 동일하게 `${styles.advSearchWrap} ${styles.advSearchNarrow}` wrapper로 감싸 세로 중앙 정렬되도록 수정

주요 결정:
- 새 CSS를 추가하지 않고 이미 있는 `advSearchWrap`/`advSearchNarrow`(세로 중앙 정렬 + 고정 너비)를 재사용 — 작성자 행과 시각적으로도 통일됨
- 검증: `tsc --noEmit`·`eslint` 클린, curl로 정상 렌더링 확인

## 95. 검색 위치를 요청함과 통일 (헤더 안으로 이동)

사용자 요청: "문서함의 검색 위치를 요청함에 있는 위치랑 맞춰줘"

문서함은 헤더(제목만)와 별도로 그 아래 `.toolbar` 줄에 검색창이 있었는데, 요청함은 검색창이 헤더 안(제목 옆, 태블릿+에서만 노출)에 있음. 요청함 구조 그대로 이식.

수정된 파일:
- `src/app/owner/documents/page.tsx` — `.toolbar` div를 제거하고, 검색창+검색버튼(`headerSearch`)과 세부검색 토글 버튼을 `<header>` 안, 제목 옆으로 이동
- `src/app/owner/documents/page.module.css`
  - `.heading`을 `flex:1` → `flex-shrink:0`으로 변경(요청함과 동일 — 검색 영역이 나머지 공간을 차지하도록)
  - `.headerSearch { display:none }` 추가, 태블릿+(`min-width:768px`)에서 `display:flex; align-items:center; gap:8px; flex:1; max-width:520px; margin-left:auto;`로 노출(요청함 값 그대로)
  - 더 이상 쓰이지 않는 `.toolbar` 규칙 제거(검색 관련 하위 클래스 `.searchWrap`/`.searchInput`/`.searchBtn` 등은 세부검색 패널 "내용" 행에서 계속 쓰이므로 유지)

주요 결정:
- 세부검색 패널의 "내용" 검색 인풋은 그대로 유지 — 요청함과 동일하게 헤더 검색은 모바일에서 숨겨지므로(`headerSearch`가 `display:none` 기본), 모바일에서는 "세부 검색" 패널이 검색 진입 경로 역할을 함
- 검증: `tsc --noEmit`·`eslint` 클린, curl로 헤더 영역에 "문서 검색"/"세부 검색"이 함께 렌더링됨을 확인
